AI tools write code faster than humans can review it. Without governance, every PR is a gamble — security holes, broken patterns, hallucinated logic shipping to production. Prometheus closes that gap.

The Three Fires

One package. Three layers of protection.

CLI

prometheus scan deep-scans your entire repo and generates a health grade A+ through F. prometheus review checks only staged changes — fast enough to run before every commit.

The Oracle
GitHub Action

One YAML file in your repo. Every pull request gets governance review — inline comments on the exact line, BLOCKER findings block the merge. Fully self-contained. No tokens, no secrets, no setup beyond copy-paste.

The Guardian
VS Code Extension

Real-time findings as you write. Health score in the status bar. The AI adapters panel keeps Claude, Cursor, Copilot, and Gemini aware of your governance rules — so they write governed code from line one.

Setup

Governed in minutes.
Protected forever.

No onboarding wizard. No cloud account. Prometheus works locally, in CI, and in your editor at the same time.

I

Install

Run npm install prometheus-governance. All 505 rules are active immediately — no configuration file needed.

II

Scan

Run prometheus scan to see every finding in your codebase and get your first health grade.

III

Protect CI

Copy one YAML file to .github/workflows/. Every PR from this point is governed — BLOCKER findings block the merge automatically.

IV

Extend

Install community packs or write your own. Drop into .prometheus/packs/ — they load at runtime, no rebuild needed.
