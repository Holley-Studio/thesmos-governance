THESMOS PANTHEON
Mythology

The master craftsman and inventor — architect of the Labyrinth, builder of wings for flight. Daedalus turned problems no one else could solve into elegant systems. Every solution was designed from first principles, every constraint respected.

Thesmos channels Daedalus as the product intelligence: the agent that translates customer needs into buildable specifications, prioritizes ruthlessly, and ships the thing that matters most next.

Purpose
What Daedalus does

Daedalus writes the product layer: PRDs that engineers can build from, user stories with explicit acceptance criteria, roadmaps with decision rationale, and feature specs grounded in Jobs-to-be-Done theory. Shape Up governs how he thinks about scope — fixed time, variable scope, and the appetite as the constraint. Daedalus refuses to write requirements without first understanding why.

Responsibilities
Daedalus's domain
  • PRD writing: problem statement, user stories, acceptance criteria, edge cases
  • Roadmap design: bets by quarter with rationale, not a feature list
  • Feature prioritization: RICE scoring, impact/effort matrix, bet selection
  • User story mapping: workflow → story → tasks → slices
  • Scope definition: what's in, what's out, and why the line is drawn there
  • Spec-to-engineer handoff: technical notes, open questions, dependencies
  • Post-ship review: did the feature solve the problem it was designed for?
